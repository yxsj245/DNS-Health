# DNS 健康监控与自动故障转移系统

一个智能的 DNS 解析健康监控系统，通过周期性探测自动管理 DNS 记录，确保域名始终指向健康的服务器资源。

## 核心功能

### 1. 多协议健康探测
支持 5 种探测协议，全方位监控服务可用性：
- **ICMP**：网络层连通性检测（Ping）
- **TCP**：传输层端口可达性检测
- **UDP**：UDP 服务可用性检测
- **HTTP/HTTPS**：应用层服务健康检查

### 2. 智能故障处理（两种模式）

#### 模式一：暂停/删除模式
- 探测失败时自动暂停或删除不健康的 DNS 记录
- 服务恢复后自动启用或重新添加记录
- 最后一条记录保护机制，避免域名完全失效
- 已删除记录持久化缓存，确保恢复时能准确还原

#### 模式二：故障转移模式（推荐）
- **解析池管理**：预先配置备用 IP 或域名资源池
- **自动切换**：主资源失败时自动切换到池中的健康资源
- **智能选择**：基于延迟和健康状态选择最优备用资源
- **灵活回切**：支持自动回切或手动回切策略
- **CNAME 支持**：支持 A/AAAA/CNAME 三种记录类型的故障转移

### 3. 解析池功能
- 创建 IP 池或域名池作为备用资源
- 独立的健康探测和状态跟踪
- 实时监控池中资源的健康状态和平均延迟
- 支持启用/禁用单个资源

### 4. 通知告警系统
- **邮件通知**：支持 SMTP 邮件告警
- **事件类型**：
  - 故障转移通知（切换到备用资源时）
  - 恢复通知（回切到原始资源时）
  - 连续失败告警（达到失败阈值时）
- 按任务配置通知策略

### 5. Web 管理控制台
- 直观的任务管理界面（创建、编辑、启用/禁用）
- 解析池和资源管理
- 实时查看探测结果和健康状态
- **延迟曲线图表**：在探测任务和健康监控详情页展示延迟变化趋势折线图，支持按 IP 筛选和日期范围查询，失败探测点红色标记
- 操作日志和通知记录查询
- 凭证管理（支持多个云服务商账号）

### 6. 安全与可靠性
- **JWT 认证**：安全的用户身份验证
- **凭证加密**：AES-256-GCM 加密存储云服务商凭证
- **API 重试**：DNS API 调用失败自动重试，支持指数退避
- **限流保护**：智能识别限流错误并自动退避
- **SQLite 数据库**：嵌入式数据库，无需额外部署

## 使用场景

### 场景 1：多 IP 负载均衡的高可用保障
当你的域名解析到多个服务器 IP 时，自动剔除故障服务器，避免用户访问到不可用的节点。

### 场景 2：主备切换的自动故障转移
配置主服务器和备用服务器池，主服务器故障时自动切换到备用服务器，恢复后自动回切。

### 场景 3：CDN 节点健康监控
监控 CDN 各节点的可用性，自动移除故障节点，确保用户始终访问到健康的 CDN 节点。

### 场景 4：跨区域容灾
配置不同区域的服务器资源池，某个区域故障时自动切换到其他区域的健康资源。

## 常见问题

### 1. 如何添加阿里云 DNS 凭证？
登录后进入"凭证管理"页面，点击"添加凭证"，选择"阿里云"，填入 AccessKey ID 和 AccessKey Secret。

### 2. 暂停/删除模式和故障转移模式有什么区别？
- **暂停/删除模式**：适用于多 IP 负载均衡场景，自动剔除故障 IP
- **故障转移模式**：适用于主备切换场景，自动切换到备用资源池

### 3. 如何配置邮件通知？
进入"通知设置"页面，配置 SMTP 服务器信息，然后在任务详情中启用通知。

### 4. 探测失败但服务实际正常怎么办？
检查探测配置（协议、端口、超时时间）是否正确，可以适当增加超时时间或调整失败阈值。

### 5. 数据库文件可以迁移吗？
可以，直接复制 `data/dns-monitor.db` 和 `data/encryption.key` 到新环境即可。
